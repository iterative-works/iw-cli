## Testing Review

### Critical Issues
- **InputStream not closed in `serveClasspathResource`** (`CaskServer.scala:623-624`) ‚Äî `getResourceAsStream` stream consumed via `readAllBytes()` but never closed. Resource leak on every static asset request. No test covers this and existing code elsewhere in the file (lines 285, 1198) uses `try/finally source.close()`.
- **Port-reuse race in `freePort`** (`CaskServerItest.scala:47-51`) ‚Äî opens `ServerSocket(0)`, closes, then returns port; window allows another binder. Under `testForked` parallelism, intermittent `BindException`.

### Warnings
- **Per-test server start without shared fixture** (`CaskServerItest.scala:64-213`) ‚Äî 8 HTTP tests each spin up a server with no teardown; leaked daemon threads on test failures.
- **Jar-contents tests open `ZipFile` 4x for essentially one assertion group** (`CaskServerItest.scala:238-284`) ‚Äî boilerplate duplication; consolidate.
- **`GET /static/dashboard.js` test missing body non-emptiness assertion** (`CaskServerItest.scala:123-139`).

### Suggestions
- `resolveJarPath` filesystem walk ‚Äî prefer a Mill-injected system property.
- `cleanup` could throw `DirectoryNotEmptyException` and mask test failures ‚Äî use `os.remove.all`.

</review>

## API Design Review

### Critical Issues
- **Path traversal in `serveClasspathResource`** (`CaskServer.scala:617-632`) ‚Äî `s"/static/$filename"` interpolation unvalidated; `GET /static/../META-INF/MANIFEST.MF` exposes manifest and bundled class files. Reject any `filename` containing `/`, `\`, `..`.
- **`InputStream` left open on happy path** (`CaskServer.scala:623-624`) ‚Äî wrap in `scala.util.Using.resource`.

### Warnings
- **404 has no `Content-Type` or body** ‚Äî inconsistent with JSON error bodies elsewhere (`/api/v1/worktrees/:issueId/status` uses `ujson.Obj("code","message")`). Either document the deviation or align.
- **No `Cache-Control` on static asset responses** ‚Äî Vite-hashed `/assets/*` can use `public, max-age=31536000, immutable`; `/static/*` should be `no-cache`.
- **Integration tests for 404 don't assert body** ‚Äî status-only check lets silent body leaks through.

### Suggestions
- `serveClasspathResource` accepts arbitrary paths ‚Äî constrain via `require` or sealed prefix type.
- `freePort()` TOCTOU ‚Äî see testing review.

</review>
