## Security Review

### Critical Issues

None found. No injection-class issues. `os.proc` and `GitAdapter.commit` use `Seq` semantics throughout. `WEBAWESOME_NPM_TOKEN` is consumed only by Mill/Yarn during the build step; never written to the tarball, logged, or echoed.

### Warnings

#### `IW_CORE_JAR` env override accepts any path
**Location:** `iw-run:89-91` and analogous `IW_DASHBOARD_JAR` block
**Problem:** Only `-f` and `*.jar` checks; no boundary on directory. Anyone able to set the env var can point it at arbitrary code.
**Note:** This is an intentional escape hatch for tests/CI (per Decision 3, the env var "remains the escape hatch"). Reviewer recommends documenting this trust assumption with a comment near the check.

#### `--issue-id` CLI flag bypasses `IssueId` validation
**Location:** `commands/review-state.scala:208-213`, `:380-386`
**Problem:** Branch-derived issue IDs go through `IssueId.fromBranch`, but `--issue-id <raw>` is used as a raw string, then interpolated into commit messages and into a `project-management/issues/<id>/` path. No shell injection (via `Seq`-based `os.proc`), but malformed IDs slip into permanent commits and the path could in principle traverse if the user passes `../foo`.
**Recommendation:** Route `--issue-id` through `IssueId.parse`.

#### Unquoted variable expansion in `iw-run` exec lines
**Location:** `iw-run:683`, `:768`
**Problem:** `$hook_files` and `$lib_files` are space-separated strings expanded unquoted into `exec scala-cli`. A path with a space is mis-split. Pre-existing pattern; `lib_files` is new in this diff.
**Recommendation:** Migrate file lists to bash arrays. Out of scope for this iteration; flag for a follow-up.

#### `validate_jar` does not enforce a path boundary
**Location:** `scripts/package-release.sh:36-41`
**Problem:** Only validates extension/size/existence; no `realpath`-based check that the resolved jar lives under `$PROJECT_ROOT`. In a tampered Mill output, an arbitrary file could be copied into the tarball.
**Recommendation:** Add a `realpath`-prefix check.

### Suggestions

- `strip_mill_ref` sed regex anchored only at start, not trimmed at end — a stray newline/space after the path would survive.
- `--output`/`--input` paths resolved relative to `os.pwd` with no `..` rejection (low risk for an interactive dev tool, real if invoked from CI with attacker-controlled input).
- `WEBAWESOME_NPM_TOKEN` log scrubbing is a CI-config concern, not a code concern; flag for CI review (Phase 2).

</review>
