### Critical issues (must fix)

1. **`catch case e: Throwable` swallows fatal JVM errors** — `core/commands/Rm.scala:135`. Catches `OutOfMemoryError`, `StackOverflowError`, `InterruptedException`, and Scala control exceptions, then continues on a degraded JVM. Use `scala.util.control.NonFatal`. (error-handling; also flagged Warning by scala3)
2. **`e.getMessage` can be `null`** — `core/commands/Rm.scala:136`. A hook throwing `new RuntimeException()` (no message) prints `Error: Cleanup hook failed: null`. Fall back to `e.getClass.getSimpleName`. (error-handling, security)
3. **Scenario 2 "hook ran before removal" is asserted only in a comment** — `core/test/RmHarnessTest.scala:234-236`. The test checks the hook ran AND removal happened, but not their order. Use a shared event log (as scenario 5 does) so a regression that runs the hook after removal is caught. (testing)
4. **BATS sentinel check lacks an explicit assertion** — `test/rm.bats`. If `HOOK_RAN` is absent, the later ordering check fails with a confusing message instead of a clear "sentinel not found". Add `echo "$output" | grep -q "HOOK_RAN" || fail "..."`. (testing)
